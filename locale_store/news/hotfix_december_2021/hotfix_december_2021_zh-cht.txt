Wallpaper Engine 2.0.XX：Windows 防火牆偵測與 Android 修復

Wallpaper Engine 2.0.XX 更新版本可解決與 Android 發布版本最常見的問題，我們也正在更新該應用程式的 Windows 及 Android 版本。

今天發布的更新版本可解決與 Android 發布版本最常見的問題，我們也正在更新該應用程式的 Windows 及 Android 版本。

[h2]連接 Android 及 Windows 版應用程式[/h2]
我們最常收到的問題是，欲下載更多桌布時，無法連接 Android 版及 Windows 版應用程式。 經調查，發現許多情況都是由 Windows 防火牆造成。 如果先前 Windows 請您授予 Wallpaper Engine (「ui32.exe」) 存取網路的權限，但您直接關閉視窗或當初並未開啟 Wallpaper Engine 權限，Windows 將自動阻擋 Wallpaper Engine 與您的區域網路正常通訊：

[img]{STEAM_CLAN_IMAGE}/26312799/46ccf465f7d479f9e1d305b2001e88ea2d791c49.png[/img]
[i]如果您按下「取消」或是直接關閉視窗，Wallpaper Engine 則會遭到 Windows 防火牆封鎖。[/i]

我們新增了一項檢查功能，會協助偵測此類問題，並提供自動修復 Windows 防火牆的方式。 不過如果您並未收到 Windows 防火牆任何通知，或如果您有其他防火牆應用程式，Wallpaper Engine 就無法自動修復您的區域網路。

如果您安裝了任何其他防火牆或防毒應用程式，必須確定 [b]ui32.exe[/b] 並未遭到封鎖。

Windows 上的 Wallpaper Engine 現在也會使用其他方法在您的網路上廣播，將有助在某些網路設定下的 Android 裝置更方便存取此應用程式。

如果仍有連接問題，請閱讀以下說明頁面：
[list][*] [url=https://help.wallpaperengine.io/mobile/pairing-fixes.html]Android 版應用程式連線問題疑難排解[/url][/list] 請注意，您隨時可利用匯出功能當作備案，直接透過 USB 或其他方式將桌布傳輸至行動裝置。 將桌布放在行動裝置上時，務必確定儲存在手機或平板電腦的[b]下載[/b]目錄。 為了保障安全，Android 相當嚴謹，Wallpaper Engine 無法存取每個 Android 資料夾，但下載目錄通常最不會出錯。

[h2]Android 上的鎖定畫面桌布[/h2]
由於我們常從電子郵件和論壇上收到這類要求，謹在此重申：[b]Wallpaper Engine 完全支援 Android 鎖定畫面的動態桌布。[/b]不過，許多手機和平板電腦卻不支援鎖定畫面的動態桌布。

如果您找不到設定鎖定畫面動態桌布的選項，代表手機製造商設下此限制。 Wallpaper Engine 無權管控，我們或其他任何動態桌布應用程式也無法解決此問題。遇到這種情況，最佳辦法就是向手機製造商反應，讓他們知道您希望有這項功能，請他們考慮新增至 Android 更新版本。
